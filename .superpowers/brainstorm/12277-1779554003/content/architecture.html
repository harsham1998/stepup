Key Data Flows

Step Sync Flow
1
Flutter polls HealthKit/Health Connect every 15 min in background
2
POST /steps/sync with raw step data + device metadata
3
Anti-cheat validates: steps/min rate, heart rate consistency, GPS movement
4
Valid steps → write to step_logs, update Redis ZADD for active challenges
5
Supabase Realtime pushes updated rank to all subscribers
Join Paid Challenge
1
User taps "Join · ₹50" — Razorpay SDK opens UPI/card sheet
2
Razorpay webhook → POST /wallet/deposit confirmed
3
Wallet module: debit entry fee, credit challenge pool atomically (DB transaction)
4
Insert into challenge_participants, add to Redis leaderboard set
5
Push notification: "You've joined Weekend Warriors! 47 competitors."
Challenge Payout
1
BullMQ cron job runs at challenge end time
2
Fetch final ranks from Redis, cross-check with Postgres step_logs
3
Anti-cheat final sweep: flag suspicious step patterns
4
Distribute pool to winners (top % based on challenge config), deduct 10% platform fee
5
Credit winner wallets, queue Razorpay payout for UPI withdrawals

Scale headroom at 100k users
Redis sorted sets handle 10M+ leaderboard ops/sec. Supabase Postgres scales to ~50k concurrent connections with pgBouncer. BullMQ with Redis processes 100k+ step sync jobs/hour. When one module becomes the bottleneck, extract it to its own service — the clean boundaries make this a lift-and-shift operation.
