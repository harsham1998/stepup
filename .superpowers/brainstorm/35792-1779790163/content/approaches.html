Implementation Approach

How should we structure the gamification overhaul?

A

Layered (Recommended)

4 focused layers, each independently testable:

  1. XP Engine — fix mission XP bug, add challenge-win XP, level-up coin rewards
  2. Streak v2 — calendar view, partial-activity break logic, proper 2-day grace
  3. Fitness Reputation — real calculation across 5 sub-scores, 0–900 scale
  4. Seasons — admin table, XP reset, tier soft-decay, rewards page

Pros

  • Each layer ships independently
  • Easy to test in isolation

Cons

  • 4 PRs instead of 1
B

Big Bang

All subsystems in one large PR.

  1. All API fixes and new endpoints
  2. All Flutter screen wiring
  3. All streak, reputation, season logic

Pros

  • Single coherent release
  • No half-baked states in prod

Cons

  • Very large, hard to review
  • One bug blocks everything
C

UI-First

Wire all Flutter screens to APIs first, fix backend as needed.

  1. XP level screen → real API
  2. Reputation screen → real API
  3. Streak calendar → real data
  4. Then fill in missing backend logic

Pros

  • Immediately visible in app
  • Designer-friendly workflow

Cons

  • Surfaces broken APIs quickly — needs fixing anyway
  • Can't test UI without backend
