Layer 4 — Seasons

Admin-declared periods · XP reset · tier soft-decay · end-of-season rewards page

DB SCHEMA — seasons table (new)
CREATE TABLE seasons (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL, -- "Season 1: Foundation"
  start_date DATE NOT NULL,
  end_date DATE NOT NULL,
  status TEXT NOT NULL DEFAULT 'upcoming', -- upcoming | active | ended
  tier_decay_pct INT NOT NULL DEFAULT 50, -- % of XP carried to next season
  created_at TIMESTAMPTZ DEFAULT NOW()
);

SEASON END LOGIC (runs when admin marks season 'ended')
Step 1
Snapshot each user's final tier into user_season_results table
Records: user_id, season_id, final_league_slug, final_xp, rank_in_tier
Step 2
Tier soft-decay — carry forward 50% of XP
Elite (10K XP) → keeps 5K → lands in Diamond · Diamond (6K) → 3K → Platinum
users.xp = floor(users.xp × 0.5) · user_leagues.xp updated · tier slug recalculated
Step 3
user_levels.xp stays — lifetime XP for levels & reputation is never reset
Only league XP (users.xp) decays — levels and reputation use cumulative xp
Step 4
Award season rewards — coins based on final tier
Elite: 2000 coins · Diamond: 1000 · Platinum: 500 · Gold: 250 · Silver: 100 · Bronze: 25
Step 5
Mark season 'ended', activate next season (if one exists in upcoming state)
NEW DB TABLES
CREATE TABLE user_season_results (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  season_id UUID REFERENCES seasons(id),
  final_league_slug TEXT NOT NULL,
  final_xp INT NOT NULL,
  rank_in_tier INT,
  coins_awarded INT DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, season_id)
);
API ENDPOINTS
GET /seasons/current — active season info (name, start, end, days left)
GET /seasons/:id/my-result — user's result for a past season
POST /seasons/:id/end — admin only · triggers end-of-season logic
